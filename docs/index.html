Structural Extraction Our moat

Most parsers convert Office files to PDF first, losing structure. DocParse reads the XML directly. These are the features competitors miss entirely.

Track Changes
Insert, delete, and move blocks with author attribution and timestamps. Structured ChangeBlock data, not rendered accept/reject text.
Exclusive
Merged Cells
Table headers, rows, and cells with colspan/rowspan merge info preserved. Not atomized into individual cell elements.
Exclusive
Headers & Footers
Extracted as semantic SectionBlocks with typed sub-blocks. Not flattened into body text like competitors.
Better
Text Boxes & Shapes
Content extracted from DrawingML and VML shapes, including legacy VML images that competitors silently drop.
Exclusive
Embedded Images
Images detected with base64 data and MIME type. Optional AI descriptions via Gemini, Claude, or Ollama multimodal.
Better
Contract Verified
63 ensures contracts across all modules. Filter bounds, 1:1 mapper preservation, structural invariants. Z3-verifiable at compile time.

Usage

One command. Every format. CLI or library.

Office Documents (no AI needed)
# Install git clone https://github.com/sunholo-data/docparse.git ln -s "$(pwd)/docparse/bin/docparse" /usr/local/bin/docparse # Parse any document (Office formats — instant, no AI) docparse report.docx docparse slides.pptx docparse data.xlsx docparse document.odt docparse book.epub
PDFs and Images (AI-powered)
# PDF and image extraction (auto-selects AI backend) docparse invoice.pdf docparse scan.png # Choose your AI backend docparse doc.pdf --ai gemini-2.5-flash # Google (default) docparse doc.pdf --ai claude-haiku-4-5 # Anthropic docparse doc.pdf --ai granite-docling # Local Ollama (free)
Dev Tools
docparse --check # Type-check all 18 modules docparse --test # Run 51 inline tests docparse --prove # Z3 contract verification docparse report.docx --verify # Runtime contract checks docparse report.docx --describe # AI image descriptions docparse report.docx --summarize # AI document summary
AI-Agnostic Any model

DocParse uses AILANG's AI effect system. Same code, any backend — swap --ai to change the model. AI usage is bounded by capability budgets, so costs are predictable.
