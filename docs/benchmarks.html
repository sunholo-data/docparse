Office Structural Benchmark (OfficeDocBench)

Most benchmarks focus on PDF extraction accuracy. Nobody benchmarks Office parsing quality — whether a parser preserves track changes, merged cells, comments, headers/footers, and text boxes. This is our moat.

OfficeDocBench is the first benchmark focused on structural Office parsing quality. 53 real-world files across 10 formats, checking 8 structural properties that competitors miss entirely.

PDF Benchmark (OmniDocBench)

For PDFs, DocParse delegates to whatever AI model you plug in. The benchmark measures AI accuracy, not our parsing code.

ModelText ED ↓Table TEDS ↑Reading Order ED ↓
Gemini 2.5 Flash0.1830.8710.141
Gemini 2.0 Flash0.2100.8420.168
Ollama (granite-docling)0.8900.1200.920
Recommended: gemini-2.5-flash — best balance of accuracy and speed. Ollama models score low due to structured JSON output limitations.

Run the Benchmarks

# Quick smoke test (~15 seconds)
bash benchmarks/quick_check.sh

# Full 53-file Office structural benchmark (~10 minutes)
uv run benchmarks/run_benchmarks.py --suite office

# PDF benchmark (needs AI model)
uv run benchmarks/run_benchmarks.py --suite pdf --ai gemini-2.5-flash

# Competitor comparison
uv run benchmarks/run_benchmarks.py --competitors              # all competitors
uv run benchmarks/run_benchmarks.py --competitors docling      # just Docling
uv run benchmarks/run_benchmarks.py --competitors llamaparse   # just LlamaParse

# Regenerate golden outputs after parser changes
bash benchmarks/generate_golden.sh
